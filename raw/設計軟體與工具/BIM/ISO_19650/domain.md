# ISO 19650 資訊管理權責分工（PDCA 模式）

本文件說明如何根據 ISO 19650 國際標準，將專案中的資訊管理職責透過 PDCA（計畫-執行-檢查-行動）循環進行系統化分工，適用於台灣建築師事務所之 BIM 專案管理。

## 角色定義

在 ISO 19650 體系中，主要分為三個角色：
1. **業主 / 主辦機關 (Appointing Party)**：專案的發起者，負責定義需求。
2. **專案負責人 / 主承攬人 (Lead Appointed Party)**：如建築師事務所或總包商，負責協調整體資訊交付。
3. **協力廠商 / 協作單位 (Appointed Party)**：如機電技師、結構技師或專業分包，負責生產具體資訊。

## PDCA 權責分工表

依據 ISO 19650-2 的八大階段映射至 PDCA 循環：

| PDCA 階段 | ISO 19650 階段 | 核心任務 | 業主 (AP) | 主承攬人 (LAP) | 協作單位 (Apty) |
| :--- | :--- | :--- | :---: | :---: | :---: |
| **P (Plan)** | **1. 需求評估** | 建立 PIR、專案標準與里程碑 | **A/R** | C | I |
| | **2. 招標邀請** | 編製 EIR 與資訊協議 (Protocol) | **A/R** | C | I |
| | **3. 投標回應** | 擬定初步 BEP、能力評估 | I | **A/R** | C |
| **D (Do)** | **4. 合約委任** | 確認正式 BEP、MIDP 與 TIDP | **A** | **R** | C |
| | **5. 動員** | 測試 CDE、確認生產流程 | I | **A/R** | **R** |
| | **6. 協作生產** | 資訊模型生產、碰撞檢查、協調 | I | **A** | **R** |
| **C (Check)** | **7. 交付審核** | 模型核發與驗收 | **A/R** | **R** | I |
| **A (Act)** | **8. 專案結案** | 經驗學習、資訊歸檔至 AIM | **A** | **R** | C |

*註：R (Responsible)-執行者, A (Accountable)-決策/負責者, C (Consulted)-諮詢者, I (Informed)-被通知者*

## 實務應用重點

### 1. 資訊需求層級 (Information Requirements)
- **PIR (專案資訊需求)**：業主應明確定義專案目標。
- **EIR (交換資訊需求)**：建築師應協助業主將 PIR 轉化為技術性的招標要求。

### 2. BIM 執行計畫 (BEP)
- **投標前 BEP**：展示團隊有能力達成要求。
- **投標後 BEP**：詳細說明「誰、在何時、如何」生產資訊。

### 3. 交付計畫 (MIDP & TIDP)
- **MIDP (總資訊交付計畫)**：由建築師彙整所有技師的 TIDP，確保時程不衝突。
- **TIDP (任務資訊交付計畫)**：各專業團隊針對其負責範圍擬定的詳細交付時程。
