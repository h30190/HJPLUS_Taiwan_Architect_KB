# ISO 19650 Responsibility Matrix (PDCA Model)

This skill provides a structured framework for allocating information management responsibilities during the delivery phase of assets, following the ISO 19650-2 standard and organized by the PDCA cycle.

## Information Management Stages & PDCA Mapping:

| PDCA Phase | ISO 19650-2 Stage | Key Activities |
|------------|-------------------|----------------|
| **Plan** | 1. Assessment and Need | Establish project information requirements (PIR), milestones, and standards. |
| **Plan** | 2. Invitation to Tender | Prepare project information protocol and tender response requirements. |
| **Plan** | 3. Tender Response | Lead Appointed Party prepares BIM Execution Plan (BEP) and capability assessment. |
| **Do** | 4. Appointment | Finalize BEP, Master Information Delivery Plan (MIDP), and Task Information Delivery Plan (TIDP). |
| **Do** | 5. Mobilization | Test information production methods and procedures; set up Common Data Environment (CDE). |
| **Do** | 6. Collaborative Production | Production of information by task teams; coordination and spatial coordination. |
| **Check** | 7. Information Model Delivery | Review and authorize information models before delivery to the Appointing Party. |
| **Act** | 8. Project Close-out | Archive project information; capture lessons learned; transfer to Asset Information Model (AIM). |

## Responsibility Matrix (RACI):

| Role | Description | PDCA Focus |
|------|-------------|------------|
| **Appointing Party (AP)** | The Employer or Client. | Plan (Requirements), Check (Review/Approval). |
| **Lead Appointed Party (LAP)** | Tier 1 Lead Consultant or Main Contractor. | Plan (BEP), Do (Management), Check (Consolidation). |
| **Appointed Party (Apty)** | Sub-consultants or Sub-contractors. | Do (Information Production). |

## Technical Specifications:

### Information Requirements Hierarchy:
1. **OIR** (Organization Information Requirements)
2. **PIR** (Project Information Requirements)
3. **AIR** (Asset Information Requirements)
4. **EIR** (Exchange Information Requirements)

### Key Documents:
- **BEP (Pre-appointment)**: Demonstrates capability and proposed approach.
- **BEP (Post-appointment)**: Detailed execution strategy.
- **MIDP**: Master schedule for all information delivery.
- **TIDP**: Task-specific schedule for individual teams.
