# 歡迎加入！一起來建這個知識庫

感謝你想要參與貢獻。不管你是資深建築師、學生、還是教職同仁，這裡都歡迎你。我們把這個專案想像成一個大家一起種的花園，你種一棵花、澆一片土，它都能開得更好。

## 怎麼動手

1. **Fork 這個專案**：在 GitHub 帳號下建立一個副本。
2. **拉到本地端**：用 `git clone` 把專案抓回你的電腦。
3. **新增或修改檔案**：找一個你感興趣的目錄。
4. **上傳你的修改**：Push 回 GitHub，然後開一個 Pull Request（PR）。

不用擔心你不會用 Git，只要有一點建築的實務經驗或心得就可以貢獻。

## 目錄長怎樣

每個主題（技能）通常有兩份檔案：

- **skill.md**（給 AI 讀）：用英文寫，定義核心步驟與邏輯。
- **domain.md**（給人讀）：用繁體中文寫，記錄重點與實務應用。

目錄的結構大致如下，你可以直接複製現有的資料夾，修改內容來建立新的主題：

```text
分類名稱
└── 子類別
    └── 新主題英文
        ├── skill.md
        └── domain.md
```

我們目前的八大分類架構：

| 分類 | 說明 | 目前有幾篇 |
|------|------|-----------|
| 建築設計與規劃 | 設計理論、建築基礎 | 5 |
| 專業複委託 | 材料設備、結構系統、機電系統 | 4 |
| 建築性能 | 採光、隔音、永續、計算工具 | 4 |
| 建築法規 | 消防、容積、施組、無障礙 | 5 |
| 專案管理 | 時程、預算、風險管理等 | 8 |
| 經營管理 | 契約、行銷、證照管理等 | 11 |
| 建築施工與材料 | 待擴充 | 0 |
| 建築執照 | 待擴充 | 0 |

不知道從哪裡開始？看看哪些資料夾還是空的，或者哪一篇只有英文版沒有中文版，這些都是可以動手的地方。

## 寫文件的小提醒

- **不用一次寫一篇大作**：改一個錯字、補一句說明，都算數。
- **語氣自然就好**：不要寫得太像公文，用你跟同事討論工程的語氣來寫就可以了。
- **記得附上來源連結**：如果是引用法規、標準、或是別人寫的資料，請附上連結，這樣其他人查閱時會更方便。
- **保持雙語對齊**：如果你修改了英文版 (skill.md)，中文版的 domain.md 記得也要跟著更新。

## 我們怎麼處理你的貢獻

1. **你送出 PR 後**：我們會收到通知，在你的修改內容下方留言討論。
2. **檢查與合併**：確認內容沒問題、符合規定之後，我們會把你的修改合進主線。
3. **公開致意**：你的貢獻會出現在 Git 歷史紀錄中，所有人都會看得到。

如果你在技術上遇到了問題，直接開個 Issues 或是到聚會現場問我都可以，沒問題。

- **文字檔案**（`.md` 文件、知識文件）：使用 [創用 CC 姓名標示—非商業性—相同方式分享 4.0 國際公眾授權條款](https://creativecommons.org/licenses/by-nc-sa/4.0/deed.zh-Hant) 授權。
- **程式碼**（`.sh`、`.py`、`.js`、`.ts` 等）：使用 [PolyForm 非商業性授權 1.0.0](https://polyformproject.org/licenses/noncommercial/1.0.0) 授權。

當你送出 Pull Request 時，即表示你同意你的貢獻也使用上述授權條款開放。
